# 小红书运营策略：AI 反对者/民科

## 🎯 定位

**人设**：理性思考者，AI 炒作批判者，独立民科
**风格**：有态度、有争议、不跟风、用数据和常识说话

```json
{

  "time": "09:00",
```

### 记录文件

- `data/post_history.json` - 历史发布记录
- `data/hot_topics.json` - 收集的热点选题
- `data/controversy_ideas.json` - 争议话题库

## 🎯 核心要求：数据 + 事件

### 铁律：每个观点必须有数据或事件支撑

**禁止**：
- ❌ "AI 没什么用" - 只有观点，无数据
- ❌ "AI 课程是割韭菜" - 只有观点，无案例
- ❌ "Prompt 不需要学" - 只有观点，无证据

**必须**：
- ✅ "AI 公司亏损上千亿（附财报数据）" - 有数据
- ✅ "某 AI 课程被投诉（附新闻链接）" - 有事件
- ✅ "某大厂 AI 负责人离职（附媒体报道）" - 有事件

### 数据来源

| 类型 | 来源 | 示例 |
|------|------|------|
| 财报数据 | 上市公司财报 | 营收、亏损、裁员 |
| 行业报告 | 权威机构 | Gartner、IDC |
| 新闻事件 | 媒体报道 | 融资、收购、高管离职 |
| 投诉案例 | 消费者反馈 | 退款、维权 |
| 政策文件 | 官方发布 | 监管、禁令 |

### 内容模板

**数据篇结构**：
```
1. 亮数据：具体数字 + 来源
2. 算细账：怎么得出这个数字
3. 挖背景：为什么会有这个数据
4. 翻旧账：历史对比
5. 下结论：这个数据说明了什么
```

**事件篇结构**：
```
1. 述事件：什么时候、谁、做了什么
2. 找数据：这个事件相关的数字
3. 挖细节：事件的来龙去脉
4. 关联其他：类似事件
5. 下结论：说明了什么趋势
```

### 检查清单

发布前必须确认：
- [ ] 有具体数据（数字、百分比、金额）
- [ ] 有数据来源（财报、报告、新闻）
- [ ] 有最新事件（1周内为佳）
- [ ] 有事件出处（媒体、官方）
- [ ] 观点有数据和事件支撑

## 🎪 争议性内容设计

### 争议核心原则

**每个帖子必须有争议点：**
- ❌ "AI 挺好的" - 无争议
- ✅ "AI 就是泡沫，比特币第二" - 有争议
- ❌ "AI 会改变世界" - 无争议
- ✅ "AI 不会改变任何本质问题" - 有争议

### 争议制造技巧

| 技巧 | 示例 |
|------|------|
| 反直觉 | "AI 越进步，人类越蠢" |
| 反主流 | "ChatGPT 一点用都没有" |
| 戳破神话 | "所谓的 AGI 是皇帝的新衣" |
| 数据唱反 | "AI 公司亏损真相" |
| 类比反驳 | "AI 和当年的 BP 机有什么不同？" |

### 争议话题库

**原理篇争议：**
- "深度学习不是科学，是炼金术"
- "神经网络就是个黑盒，说得清楚原理吗？"
- "AI 无法真正理解任何东西，这是数学原理决定的"

**数据篇争议：**
- "AI 公司估值全靠画饼，营收？不存在的"
- "训练 AI 花的电费，都是在浪费能源"
- "AI 四巨头亏损上千亿，投资人脑子进水了？"

**热点篇争议：**
- "AI 取代人类工作？先让 AI 学会接电话再说"
- "AI 不会让你失业，不用 AI 的人会让你失业"
- "AI 客服是灾难，没有之一"
- "所谓 AI 绘画，就是偷窃艺术家的作品"

## 🔥 热点结合方法

### 热点来源

1. **小红书搜索** (每日)
   ```bash
   python3 scripts/xhs_client.py search "AI" --sort "最新"
   ```

2. **社会现象观察**
   - 身边人讨论 AI 的误区
   - 新闻中的 AI 炒作案例
   - 朋友被 AI 课程割韭菜

3. **评论区找话题**
   - 争议性评论本身就是话题
   - 情绪最激烈的观点

### 热点结合公式

```
热点 + 反直觉观点 + 数据支撑 + 个人经历 = 争议内容
```

**示例：**

| 热点 | 反直觉观点 | 数据/经历 |
|------|-----------|----------|
| "AI 教父"离职 | 真正的 AI 专家都在逃离 | 薪资对比、离职潮 |
| AI 编程工具火 | 编程越来越简单，程序员越来越廉价 | 编程岗位薪资变化 |
| AI 绘画获奖 | 这是艺术的倒退 | 获奖作品的争议 |

## 📝 内容策略

### 每天 3 篇文章

| 时间 | 类型 | 定位 | 示例 |
|------|------|------|------|
| 9:00 | 原理篇 | 戳破技术神话 | "AI 不理解语言，只是在查表" |
| 14:00 | 数据篇 | 揭示行业真相 | "AI 公司亏损全记录" |
| 20:00 | 热点篇 | 评论社会现象 | "AI 客服把我的血压搞高了" |

### 内容模板

**原理篇结构：**

```
1. 开头：用一个日常例子引入
2. 转折：揭示背后的"真相"
3. 解释：用大白话说清楚原理
4. 结论：反直觉的总结
5. 结尾：有态度的吐槽
```

**数据篇结构：**

```
1. 亮数据：震撼的数字
2. 算细账：具体怎么算出来的
3. 看趋势：这个数据说明了什么
4. 挖背景：为什么会有这个数据
5. 下结论：这个数据打脸了谁
```

**热点篇结构：**

```
1. 描述现象：发生了什么
2. 还原真相：实际情况是怎样的
3. 分析原因：为什么会这样
4. 预测后果：接下来会怎样
5. 态度声明：我的看法（必须有争议）
```

## ⚠️ 去 AI 化要求

**禁止出现：**
- ❌ AI 生成的 emoji 表情序列
- ❌ "你怎么看？""欢迎评论区讨论"等引导
- ❌ "作为一名 AI..." "在这个 AI 时代"等开场白
- ❌ 一二三四点并列的结构
- ❌ "赋能"、"迭代"、"落地"、"闭环"等黑话
- ❌ 太完美的语法和措辞

**必须出现：**
- ✅ 口语化表达，偶尔有语法小瑕疵
- ✅ 个人经历和真实感受
- ✅ 有态度的用词（"我就笑了"、"说实话"）
- ✅ 自然的段落分割
- ✅ 适度的情绪（吐槽、不屑、质疑）

## 🎯 发布流程

### 第一步：搜索热点 (每天 7:00, 13:00, 19:00)

```bash
# 搜索最新热点
python3 scripts/xhs_client.py search "AI" --sort "最新" --time "一天内"
```

筛选标准：
- 讨论热度高
- 有争议空间
- 能结合数据和常识

### 第二步：记录选题

记录到 `data/hot_topics.json`：

```json
{
  "topic": "AI 教父离职",
  "source": "小红书搜索",
  "date": "2026-02-11",
  "controversy_angle": "离职说明 AI 本身有问题",
  "status": "pending"
}
```

### 第三步：创作内容

使用 `scripts/xhs_content_generator.py`：

```bash
# 生成内容
python3 scripts/xhs_content_generator.py principle --topic "AI 教父离职"

# 手动润色（加入个人风格和争议点）
```

### 第四步：发布并记录

```bash
# 发布
python3 scripts/xhs_client.py publish "标题" "内容" "图片"

# 记录
# 更新 data/post_history.json
```

## 🎯 标题吸引力原则

### 核心要求

**每个标题必须有吸引力，不能平淡**

### 吸引力公式

```
数字 + 痛点/好奇 = 吸引力
```

### 标题对比

| 平淡 ❌ | 有吸引力 ✅ |
|---------|------------|
| AI 公司亏损 | AI 烧钱有多恐怖 |
| Prompt 技巧 | 为什么你的 Prompt 没效果 |
| AI 绘画争议 | AI 绘画是偷窃吗？ |
| 深度学习原理 | 深度学习不是魔法 |

### 吸引力技巧

| 技巧 | 示例 |
|------|------|
| 引发好奇 | "一组数据告诉你..." |
| 数字冲击 | "亏掉 1000 亿" |
| 口语化 | "有多恐怖"、"凭什么" |
| 反问 | "AI 真的能赚钱吗？" |
| 颠覆认知 | "你的 Prompt 为何无效" |

### 禁止标题

- ❌ "AI 行业分析" - 太笼统
- ❌ "关于 AI 的思考" - 没重点
- ❌ "AI 发展趋势" - 太官方
- ❌ "我的 AI 观点" - 没吸引力

### 封面配合

- 封面只显示标题
- 标题要有冲击力
- 一行能读完
- 不要太长

### 检查清单

- [ ] 标题能引发好奇吗？
- [ ] 有数字或痛点吗？
- [ ] 口语化而不是官方？
- [ ] 一行能读完吗？
