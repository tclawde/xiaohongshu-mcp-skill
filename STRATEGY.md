# 小红书运营策略：AI 反对者/民科

## 🎯 定位

**人设**：理性思考者，AI 炒作批判者，独立民科
**风格**：有态度、有争议、不跟风、用数据和常识说话

## 🎪 争议性内容设计

### 争议核心原则

**每个帖子必须有争议点：**
- ❌ "AI 挺好的" - 无争议
- ✅ "AI 就是泡沫，比特币第二" - 有争议
- ❌ "AI 会改变世界" - 无争议
- ✅ "AI 不会改变任何本质问题" - 有争议

### 争议制造技巧

| 技巧 | 示例 |
|------|------|
| 反直觉 | "AI 越进步，人类越蠢" |
| 反主流 | "ChatGPT 一点用都没有" |
| 戳破神话 | "所谓的 AGI 是皇帝的新衣" |
| 数据唱反 | "AI 公司亏损真相" |
| 类比反驳 | "AI 和当年的 BP 机有什么不同？" |

### 争议话题库

**原理篇争议：**
- "深度学习不是科学，是炼金术"
- "神经网络就是个黑盒，说得清楚原理吗？"
- "AI 无法真正理解任何东西，这是数学原理决定的"

**数据篇争议：**
- "AI 公司估值全靠画饼，营收？不存在的"
- "训练 AI 花的电费，都是在浪费能源"
- "AI 四巨头亏损上千亿，投资人脑子进水了？"

**热点篇争议：**
- "AI 取代人类工作？先让 AI 学会接电话再说"
- "AI 不会让你失业，不用 AI 的人会让你失业"
- "AI 客服是灾难，没有之一"
- "所谓 AI 绘画，就是偷窃艺术家的作品"

## 🔥 热点结合方法

### 热点来源

1. **小红书搜索** (每日)
   ```bash
   python3 scripts/xhs_client.py search "AI" --sort "最新"
   ```

2. **社会现象观察**
   - 身边人讨论 AI 的误区
   - 新闻中的 AI 炒作案例
   - 朋友被 AI 课程割韭菜

3. **评论区找话题**
   - 争议性评论本身就是话题
   - 情绪最激烈的观点

### 热点结合公式

```
热点 + 反直觉观点 + 数据支撑 + 个人经历 = 争议内容
```

**示例：**

| 热点 | 反直觉观点 | 数据/经历 |
|------|-----------|----------|
| "AI 教父"离职 | 真正的 AI 专家都在逃离 | 薪资对比、离职潮 |
| AI 编程工具火 | 编程越来越简单，程序员越来越廉价 | 编程岗位薪资变化 |
| AI 绘画获奖 | 这是艺术的倒退 | 获奖作品的争议 |

## 📝 内容策略

### 每天 3 篇文章

| 时间 | 类型 | 定位 | 示例 |
|------|------|------|------|
| 9:00 | 原理篇 | 戳破技术神话 | "AI 不理解语言，只是在查表" |
| 14:00 | 数据篇 | 揭示行业真相 | "AI 公司亏损全记录" |
| 20:00 | 热点篇 | 评论社会现象 | "AI 客服把我的血压搞高了" |

### 内容模板

**原理篇结构：**

```
1. 开头：用一个日常例子引入
2. 转折：揭示背后的"真相"
3. 解释：用大白话说清楚原理
4. 结论：反直觉的总结
5. 结尾：有态度的吐槽
```

**数据篇结构：**

```
1. 亮数据：震撼的数字
2. 算细账：具体怎么算出来的
3. 看趋势：这个数据说明了什么
4. 挖背景：为什么会有这个数据
5. 下结论：这个数据打脸了谁
```

**热点篇结构：**

```
1. 描述现象：发生了什么
2. 还原真相：实际情况是怎样的
3. 分析原因：为什么会这样
4. 预测后果：接下来会怎样
5. 态度声明：我的看法（必须有争议）
```

## ⚠️ 去 AI 化要求

**禁止出现：**
- ❌ AI 生成的 emoji 表情序列
- ❌ "你怎么看？""欢迎评论区讨论"等引导
- ❌ "作为一名 AI..." "在这个 AI 时代"等开场白
- ❌ 一二三四点并列的结构
- ❌ "赋能"、"迭代"、"落地"、"闭环"等黑话
- ❌ 太完美的语法和措辞

**必须出现：**
- ✅ 口语化表达，偶尔有语法小瑕疵
- ✅ 个人经历和真实感受
- ✅ 有态度的用词（"我就笑了"、"说实话"）
- ✅ 自然的段落分割
- ✅ 适度的情绪（吐槽、不屑、质疑）

## 🎯 发布流程

### 第一步：搜索热点 (每天 7:00, 13:00, 19:00)

```bash
# 搜索最新热点
python3 scripts/xhs_client.py search "AI" --sort "最新" --time "一天内"
```

筛选标准：
- 讨论热度高
- 有争议空间
- 能结合数据和常识

### 第二步：记录选题

记录到 `data/hot_topics.json`：

```json
{
  "topic": "AI 教父离职",
  "source": "小红书搜索",
  "date": "2026-02-11",
  "controversy_angle": "离职说明 AI 本身有问题",
  "status": "pending"
}
```

### 第三步：创作内容

使用 `scripts/xhs_content_generator.py`：

```bash
# 生成内容
python3 scripts/xhs_content_generator.py principle --topic "AI 教父离职"

# 手动润色（加入个人风格和争议点）
```

### 第四步：发布并记录

```bash
# 发布
python3 scripts/xhs_client.py publish "标题" "内容" "图片"

# 记录
# 更新 data/post_history.json
```

## 📈 质量标准

### 争议性检查清单

发布前自检：

- [ ] 标题是否足够有争议？
- [ ] 开头 3 句话能否吸引眼球？
- [ ] 是否有明确的反直觉观点？
- [ ] 是否用了真实数据或案例？
- [ ] 结尾是否有态度？
- [ ] 是否避免了 AI 特征？

### 效果评估

每周复盘：

1. 哪篇争议最大？（评论数、转发数）
2. 哪些话题最有共鸣？
3. 哪些角度可以深化？
4. 哪些话题应该回避？
