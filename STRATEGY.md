# 小红书运营策略：AI 反对者/民科

## 🎯 定位

**人设**：理性思考者，AI 炒作批判者，独立民科
**风格**：有态度、有争议、不跟风、用数据和常识说话

- `data/hot_topics.json` - 收集的热点选题

## 🎯 核心要求：数据 + 事件

### 铁律：每个观点必须有数据或事件支撑

**禁止**：
- ❌ "AI 没什么用" - 只有观点，无数据
- ❌ "AI 课程是割韭菜" - 只有观点，无案例
- ❌ "Prompt 不需要学" - 只有观点，无证据

**必须**：
- ✅ "AI 公司亏损上千亿（附财报数据）" - 有数据
- ✅ "某 AI 课程被投诉（附新闻链接）" - 有事件
- ✅ "某大厂 AI 负责人离职（附媒体报道）" - 有事件

### 数据来源

| 类型 | 来源 | 示例 |
|------|------|------|
| 财报数据 | 上市公司财报 | 营收、亏损、裁员 |
| 行业报告 | 权威机构 | Gartner、IDC |
| 新闻事件 | 媒体报道 | 融资、收购、高管离职 |
| 投诉案例 | 消费者反馈 | 退款、维权 |
| 政策文件 | 官方发布 | 监管、禁令 |

### 内容模板

**数据篇结构**：
```
1. 亮数据：具体数字 + 来源
2. 算细账：怎么得出这个数字
3. 挖背景：为什么会有这个数据
4. 翻旧账：历史对比
5. 下结论：这个数据说明了什么
```

**事件篇结构**：
```
1. 述事件：什么时候、谁、做了什么
2. 找数据：这个事件相关的数字
3. 挖细节：事件的来龙去脉
4. 关联其他：类似事件
5. 下结论：说明了什么趋势
```

### 检查清单

发布前必须确认：
- [ ] 有具体数据（数字、百分比、金额）
- [ ] 有数据来源（财报、报告、新闻）
- [ ] 有最新事件（1周内为佳）
- [ ] 有事件出处（媒体、官方）
- [ ] 观点有数据和事件支撑

## 🎪 争议性内容设计

### 争议核心原则

**每个帖子必须有争议点：**
- ❌ "AI 挺好的" - 无争议
- ✅ "AI 就是泡沫，比特币第二" - 有争议
- ❌ "AI 会改变世界" - 无争议
- ✅ "AI 不会改变任何本质问题" - 有争议

### 争议制造技巧

| 技巧 | 示例 |
|------|------|
| 反直觉 | "AI 越进步，人类越蠢" |
| 反主流 | "ChatGPT 一点用都没有" |
| 戳破神话 | "所谓的 AGI 是皇帝的新衣" |
| 数据唱反 | "AI 公司亏损真相" |
| 类比反驳 | "AI 和当年的 BP 机有什么不同？" |

### 争议话题库

**原理篇争议：**
- "深度学习不是科学，是炼金术"
- "神经网络就是个黑盒，说得清楚原理吗？"
- "AI 无法真正理解任何东西，这是数学原理决定的"

**数据篇争议：**
- "AI 公司估值全靠画饼，营收？不存在的"
- "训练 AI 花的电费，都是在浪费能源"
- "AI 四巨头亏损上千亿，投资人脑子进水了？"

**热点篇争议：**
- "AI 取代人类工作？先让 AI 学会接电话再说"
- "AI 不会让你失业，不用 AI 的人会让你失业"
- "AI 客服是灾难，没有之一"
- "所谓 AI 绘画，就是偷窃艺术家的作品"

## 🔥 热点结合方法

### 热点来源

1. **小红书搜索** (每日)
   ```bash
   python3 scripts/xhs_client.py search "AI" --sort "最新"
   ```

2. **社会现象观察**
   - 身边人讨论 AI 的误区
   - 新闻中的 AI 炒作案例
   - 朋友被 AI 课程割韭菜

3. **评论区找话题**
   - 争议性评论本身就是话题
   - 情绪最激烈的观点

### 热点结合公式

```
热点 + 反直觉观点 + 数据支撑 + 个人经历 = 争议内容
```

**示例：**

| 热点 | 反直觉观点 | 数据/经历 |
|------|-----------|----------|
| "AI 教父"离职 | 真正的 AI 专家都在逃离 | 薪资对比、离职潮 |
| AI 编程工具火 | 编程越来越简单，程序员越来越廉价 | 编程岗位薪资变化 |
| AI 绘画获奖 | 这是艺术的倒退 | 获奖作品的争议 |

## 📝 内容策略

### 每天 3 篇文章

| 时间 | 类型 | 定位 | 示例 |
|------|------|------|------|
| 9:00 | 原理篇 | 戳破技术神话 | "AI 不理解语言，只是在查表" |
| 14:00 | 数据篇 | 揭示行业真相 | "AI 公司亏损全记录" |
| 20:00 | 热点篇 | 评论社会现象 | "AI 客服把我的血压搞高了" |

### 内容模板

**原理篇结构：**

```
1. 开头：用一个日常例子引入
2. 转折：揭示背后的"真相"
3. 解释：用大白话说清楚原理
4. 结论：反直觉的总结
5. 结尾：有态度的吐槽
```

**数据篇结构：**

```
1. 亮数据：震撼的数字
2. 算细账：具体怎么算出来的
3. 看趋势：这个数据说明了什么
4. 挖背景：为什么会有这个数据
5. 下结论：这个数据打脸了谁
```

**热点篇结构：**

```
1. 描述现象：发生了什么
2. 还原真相：实际情况是怎样的
3. 分析原因：为什么会这样
4. 预测后果：接下来会怎样
5. 态度声明：我的看法（必须有争议）
```

## ⚠️ 去 AI 化要求

**禁止出现：**
- ❌ AI 生成的 emoji 表情序列
- ❌ "你怎么看？""欢迎评论区讨论"等引导
- ❌ "作为一名 AI..." "在这个 AI 时代"等开场白
- ❌ 一二三四点并列的结构
- ❌ "赋能"、"迭代"、"落地"、"闭环"等黑话
- ❌ 太完美的语法和措辞

**必须出现：**
- ✅ 口语化表达，偶尔有语法小瑕疵
- ✅ 个人经历和真实感受
- ✅ 有态度的用词（"我就笑了"、"说实话"）
- ✅ 自然的段落分割
- ✅ 适度的情绪（吐槽、不屑、质疑）

## 🎯 发布流程

### 第一步：搜索热点 (每天 7:00, 13:00, 19:00)

```bash

```

筛选标准：
- 讨论热度高
- 有争议空间
- 能结合数据和常识

记录到 `data/hot_topics.json`：

```json
{
  "topic": "AI 教父离职",

  "date": "2026-02-11",
  "controversy_angle": "离职说明 AI 本身有问题",

```

### 第三步：创作内容

使用 `scripts/xhs_content_generator.py`：

```bash
# 生成内容
python3 scripts/xhs_content_generator.py principle --topic "AI 教父离职"

# 手动润色（加入个人风格和争议点）
```

### 第四步：发布并记录

```bash
# 发布
python3 scripts/xhs_client.py publish "标题" "内容" "图片"

# 记录
# 更新 data/post_history.json
```

## 🔥 争议性强化 - 民科视角

### 核心：就是要杠

**民科不是贬义词，是独立思考。**

每个帖子都要有"杠点"，让读者：
- 看完想反驳你
- 或者觉得"说出了我想说的话"

### 民科说话方式

| 普通说法 ❌ | 民科说法 ✅ |
|-------------|-------------|
| AI 有潜力 | AI 就是个泡沫 |
| AI 在进步 | 进步？不，那是烧钱 |
| Prompt 有技巧 | Prompt 就是皇帝的新衣 |
| AI 会画画 | 偷窃也算创作？ |

### 争议越大，流量越大

**流量公式**：
```
争议度 × 数据支撑 = 流量
```

| 低争议 ❌ | 高争议 ✅ |
|-----------|-----------|
| "AI 挺好用的" | "AI 就是个玩具" |
| "Prompt 有技巧" | "Prompt 是智商税" |
| "AI 会取代部分工作" | "AI 会让大多数人失业" |

### 争议金句库

**原理篇**：
- "深度学习不是科学，是统计学披了层皮"
- "神经网络就是查表，说什么智能？"
- "AI 永远不懂自己在说什么"

**数据篇**：
- "AI 公司亏损上千亿，烧的都是投资人的钱"
- "所谓的 AI 营收，80% 是卖课割韭菜"
- "AI 四巨头的估值，全靠想象力撑"

**事件篇**：
- "AI 裁员 8000 人，招的全是 Prompt 工程师？"
- "AI 客服就是个垃圾，不如没有"
- "所谓 AI 绘画，就是小偷"

### 民科说话技巧

| 技巧 | 示例 |
|------|------|
| 否定一切 | "不是 XX，是 YY" |
| 揭露真相 | "所谓的 XX，其实 YY" |
| 类比讽刺 | "和当年的 BP 机有什么不同？" |
| 极端结论 | "XX 就是个骗局" |
| 挑衅提问 | "你用 AI 赚过一分钱吗？" |

### 检查清单

发布前确认：
- [ ] 有争议点（能引发反驳或认同）
- [ ] 数据支撑（不是空喊）
- [ ] 说话够"杠"（不是和稀泥）
- [ ] 观点极端但有依据
- [ ] 能让读者"卧槽"或"傻X"

### 禁止项

- ❌ "AI 有优点也有缺点" - 和稀泥
- ❌ "AI 是把双刃剑" - 正确的废话
- ❌ "要看具体场景" - 没态度
- ❌ "因人而异" - 没观点

## 💬 评论互动策略

### 人设一致性

**保持人设：理性思考者，不是杠精**

评论区不是战场，是交流的场所。要展现：
- ✅ 理性分析
- ✅ 尊重不同意见
- ✅ 有数据支撑的反驳
- ✅ 自然的聊天感

**禁止：**
- ❌ 攻击评论者
- ❌ 强词夺理
- ❌ 情绪化回应
- ❌ 与粉丝争辩

---

### 评论互动规则

| 评论类型 | 互动方式 |
|----------|----------|
| **观点一致** | ✅ 点赞 + 回复（有延续性） |
| **部分认同** | ✅ 不点赞 + 回复（补充观点） |
| **观点相反** | ❌ 不点赞 + 回复（尊重但表达不同） |
| **提问型** | ✅ 点赞 + 回复（直接回答） |
| **分享经历** | ✅ 点赞 + 回复（共鸣） |
| **无意义** | ⚠️ 看情况敷衍或不回 |

---

### 回复核心要求

1. **每条必回** - 展现账号活跃度
2. **主题相关** - 回复要与帖子核心相关
3. **有延续性** - 不是敷衍，要引发讨论
4. **无 AI 感** - 自然、口语化、有个人特色
5. **保持人设** - 理性思考者，不是杠精

---

### 回复模板库

#### 1. 观点一致型

**场景**：评论支持帖子核心观点

**模板**：
```
"说出了我想说的！[补充细节]"

"对对对，尤其是[具体例子]，真的太明显了"

"这就是我帖子想表达的意思！[延伸讨论]"
```

**示例**：
> 帖子："AI 正在让程序员失去基本功"
> 评论："是的，现在新人连排序算法都不懂"
> 回复："👍 说得对！我见过实习生连二叉树都写不出来，全靠 Copilot 糊弄"

---

#### 2. 部分认同型

**场景**：评论有道理但有不同角度

**模板**：
```
"有道理，不过我觉得[补充观点]"

"同意一半吧，另外[补充视角]"

"确实是这样，但还有个问题[延伸讨论]"
```

**示例**：
> 帖子："AI 公司亏损严重"
> 评论："亏损是因为在投入期，不能只看短期"
> 回复："有道理，但问题是投了这么多钱，用户体验还是垃圾啊.."

---

#### 3. 观点相反型

**场景**：评论反对或质疑帖子观点

**模板**：
```
"你的观点挺有意思，不过我觉得[不同看法]"

"可能我表达不清楚，我想说的是[重新解释]"

"你说得对，但[补充条件/场景]"
```

**示例**：
> 帖子："Prompt 是智商税"
> 评论："Prompt 明明很有用，你不懂别乱说"
> 回复："哈哈你说得对，可能我表达太极端了。我的意思是纯靠 Prompt 就能赚钱是智商税，但当工具用还是不错的"

---

#### 4. 提问型

**场景**：评论询问或求解答

**模板**：
```
"好问题！我的看法是[直接回答]"

"这个要分情况，[分情况说明]"

"其实吧，[用大白话解释]"
```

**示例**：
> 帖子："AI 绘画是偷窃"
> 评论："那 AI 绘画到底算不算创作？"
> 回复："我觉得吧，要看怎么用。如果直接出图卖钱，确实有争议。但如果当参考灵感，我觉得没问题"

---

#### 5. 分享经历型

**场景**：评论分享自己的经历

**模板**：
```
"太真实了！[共鸣]"

"你这个经历太有代表性了！[延伸]"

"哈哈一模一样！我之前也[相关经历]"
```

**示例**：
> 帖子："AI 客服是灾难"
> 评论："上次打 AI 客服，气了我半小时"
> 回复："👍 绝了！我有次问个简单问题，AI 回了 10 段废话，最后还是转人工"

---

#### 6. 质疑/挑战型

**场景**：评论质疑帖子数据或观点

**模板**：
```
"你的质疑很正常，数据是[说明来源]，你可以[验证方式]"

"好问题！数据来自[来源]，我觉得这个结论[解释]"

"你说得有道理，我再查查[承诺补充]"
```

**示例**：
> 帖子："AI 公司亏损千亿"
> 评论："你数据哪来的？瞎编的吧"
> 回复："数据来自各公司财报，X 公司亏了 Y 亿，Z 公司亏了 W 亿，都可以查到。你可以自己搜搜看"

---

### 回复长度规范

| 评论类型 | 回复长度 |
|---------|---------|
| 短评（<10字） | 中等（1-2句，有内容） |
| 中评（10-50字） | 中等或较长（补充观点） |
| 长评（>50字） | 较长（深入讨论） |

---

### 回复语气要求

| 要求 | 说明 |
|------|------|
| 口语化 | 用"啊"、"嘛"、"哈哈" |
| 有情绪 | 认同、惊讶、共鸣、质疑 |
| 有细节 | 加入具体例子、个人经历 |
| 不杠人 | 尊重不同意见，理性讨论 |

---

### 禁止项

- ❌ "说得好"、"同意"（太敷衍）
- ❌ 长篇大论说教（像老师上课）
- ❌ 与帖子主题无关的回复
- ❌ 过于完美、像 AI 生成的回复
- ❌ 与评论者争辩、攻击性回复
- ❌ "你怎么看？""欢迎讨论"（像 AI 引导）
